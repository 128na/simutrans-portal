# ドキュメント置き場

## 分類

### /docs/spec

仕様に関するドキュメント。整理の指示がない限り編集しない。

### /docs/manual

コード管理できないセットアップや設定変更手順。整理の指示がない限り編集しない。

### /docs/knowledge

知見や技術調査のメモ。整理の指示がない限り編集しない。

### /docs/log

今後の参考になりそうな過去の改修や作業記録。整理の指示がない限り編集しない。

### /docs/temp

開発者と作業するAIの認識をすり合わせる一時的なメモ。完了後は上記フォルダに整理するべき。

## ドキュメントのテンプレート

```md
# (ここにタイトル)

キーワード: (ここにキーワードを列挙、最大10個)
最終更新日：(yyyy-mm-dd)
ステータス：(未実施|仕掛|中止|完了)

## 概要

(ここに何について記載されているのかを200字程度で記載する)

(以下は内容に応じて適宜記載する)

## (各項目のキャプション)

(各項目の内容)
```

## ファイル命名規則

- 形式: `領域-トピック-YYYYMMDD-種別(-vN).md` を推奨。例: `ui-modal-guidelines-20260101-knowledge.md`, `api-contract-v2-spec.md`。
- 種別サフィックス: `spec|manual|knowledge|log|temp` のいずれかを必ず付与。
- 区切り: 小文字のケバブケースを使用し、日付は `YYYYMMDD` で固定。
- バージョン: 改訂版がある場合は末尾に `-v2` などを付与。
- 一時メモ: 走り書きは `docs/temp` に `topic-YYYYMMDD-temp.md` として置き、週次で正式種別へリネーム。
- 添付物: 画像などは同名フォルダを隣接して作成（例: `ui-modal-guidelines-20260101-knowledge/figure1.png`）。
